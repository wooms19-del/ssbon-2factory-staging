## 🔑 핵심 도메인 룰 (반드시 준수)

### 시간 형식 (확정)
1. `thawing.date` = **end 날짜 = 작업일 = 박스 출고일**
2. `thawing.start`, `end` 모두 **datetime 형식** `YYYY-MM-DD HH:MM`
3. **방혈 시작 → 방혈 종료 ≥ 16시간** (룰)
4. **카트의 마지막 박스 `rfEnd` = `thawing.start`** (일관성)
5. 새 thawing 저장: `start: tod()+' '+startTime`, `date: addDays(tod(),1)`, `end: ''`
6. 전처리 시작 시 thawing.end 자동 채움 (`preprocess_v2.js` line 401-412): `ppStartFull = today + ' ' + d.start`

### 레토르트 룰 (정정된 버전, 절대 준수)
- 레토르트 1대 = 한 회차 최대 **4대차** (8대차 절대 아님)
- 레토르트 **3대** 운영, 한 회차 **150분**
- 대차당 EA: 시그니처 130g = **800EA/대차**, FC 장조림 3KG = **96EA/대차**
- "FP 보통 4대차 풀로 채워서 돌린다" = 한 회차 정원 4대차

### 수율 측정
- 회사 표준 수율 지표는 항상 **원물 대비 누적 수율** (stage-to-stage 아님)
- ERP 화면의 "해동(thawing)" = 내부 시스템의 "전처리(preprocessing)"

### 일별실적 / 월간생산량
- **일별실적 poolKey**: product 제외, (date, type) 단위 그룹화 → rowspan 병합
- **기초 재고**: 코드 하드코딩, 화면 비표시
- **진행중 박스 (end='')**: 차감 안 됨 (정상)
