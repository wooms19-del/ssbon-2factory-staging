# 공정별 원리 및 영향 요인

## 1. 방혈 (Thawing)

### 무엇을 하는가
냉동 원물 박스를 해동하면서 동시에 잔류 혈액을 빼는 공정. 16~17시간 소요.

### 정상 범위
- **방혈 시간**: 16~17시간 (start → end)
- **방혈 후 품온**: -1℃ ~ +2℃
- **박스당 무게**: 약 20kg

### 비정상 신호와 의미
- **16시간 미만 종료** → 방혈 부족, 혈수 잔류 가능 (제품 색상 어두움, 풍미 저하)
- **18시간 이상 지연** → 작업 일정 지연, 다음 공정 압박
- **품온 +3℃ 이상** → 해동 과다, 박테리아 증식 위험
- **품온 -2℃ 이하** → 해동 부족, 전처리 시 칼 사용 어려움 → 작업 시간↑, 비가식부↑

### 추적 키워드
- `thawing.start`, `thawing.end`, `barcode.rfEnd`

## 2. 전처리 (Preprocess)

### 무엇을 하는가
원물에서 지방, 근막, 힘줄 등 비가식부를 제거하는 공정. 작업자 7명 내외.

### 정상 범위
- **수율**: 96~98% (원물 대비)
- **비가식부**: 박스당 0.5~1kg 이하 (5kg 이하면 정상)
- **작업 시간**: 시간당 100~120kg/인

### 비정상 신호와 의미

#### 비가식부 비정상 증가 (5kg 초과)
**가장 흔한 신호. 원인 추적 순서:**
1. **원물 품질 의심 (가장 가능성 높음)**
   - 공급처 변경 시 발생
   - 특정 로트 (lot) 또는 입고일에 집중
   - 해당 부위 (홍두깨, 우둔, 설도) 전체에 발생
2. **해동 이슈**
   - 방혈 시간 18시간 이상
   - 품온 비정상
3. **작업자 숙련도**
   - 신입 작업자 투입
   - 평소와 다른 인력 배치

#### 작업 시간 증가
- 인력당 생산성 80kg/h 이하 → 원물 단단함 (해동 부족) 또는 작업자 변경

### 부위별 특성
- **홍두깨**: 결 방향 명확, 근막 두꺼움 → 비가식부 평균 높음 (1~1.5kg/박스)
- **우둔**: 단단함, 근막 적음 → 비가식부 낮음 (0.5kg/박스)
- **설도**: 지방 많음 → 비가식부 1~2kg/박스 (지방 제거 위주)

### 추적 키워드
- `preprocess.inKg`, `preprocess.outKg`, `inedible (비가식부)`

## 3. 자숙 (Cooking)

### 무엇을 하는가
원물을 삶아 단백질 변성 + 수분 제거. 4시간 내외, 2인 작업.

### 정상 범위
- **자숙 수율**: 50~58% (전처리 산출물 대비 50% 정도 감소)
- **자숙 손실률**: 42~50% (수분 + 지방 손실)
- **자숙 시간**: 회차당 4시간

### 비정상 신호와 의미
- **수율 50% 미만 (=손실 50% 초과)**
   - 원물 자체에 수분 많음 (해동 과다)
   - 자숙 시간 과다 → 단백질 과변성, 식감 거칠어짐
   - 온도 비정상 (95℃ 미만 또는 100℃ 초과 시 영향)
- **수율 60% 초과**
   - 자숙 부족 → 미생물 위험, 다음 공정 파쇄 어려움
   - 자숙 시간 단축 의심

### 추적 키워드
- `cooking.inKg`, `cooking.outKg`

---

## 4. 파쇄 (Shredding)

### 무엇을 하는가
자숙된 고기를 잘게 찢어 시그니처/FC 장조림용 결을 만드는 공정. 단일 라인.

### 정상 범위
- **파쇄 수율**: 92~98% (자숙 산출 대비)
- **비가식부**: 1kg 이하/회차 (자숙 시 누락된 근막, 작은 뼈 등)
- **인원**: 9~15명 (시간대별 변동)

### 비정상 신호와 의미
- **파쇄 시 다량 비가식부 (5kg 이상)**
   - 자숙 부족 → 결이 안 풀려 파쇄 시 손실
   - 또는 전처리 단계에서 근막 제거 부족
- **수율 90% 미만**
   - 작업자 손실 또는 위 두 원인

### 추적 키워드
- `shredding.inKg`, `shredding.outKg`, `shredding.inedible`

---

## 5. 포장 (Packing)

### 무엇을 하는가
파쇄 산출물을 파우치에 담아 밀봉. 시그니처 130g, FC 장조림 3KG 등.

### 정상 범위
- **포장 수율**: 105~110% (소스 첨가로 무게 증가)
- **불량률**: 1% 이하
- **인원**: 6~10명

### 비정상 신호와 의미
- **불량률 2% 이상**
   - 소비기한 판독 실패 (잉크/프린터 이슈)
   - 진공 실링 불량 (기계 점검 필요)
   - 무게 미달/초과 (저울 보정)
- **무게 미달 다발**
   - 작업자 손 무게 감각 오차
   - 작은 컵으로 변경 검토

### 추적 키워드
- `packing.ea`, `packing.defPouch`, 불량 사유

---

## 6. 외포장 (Outerpacking)

### 무엇을 하는가
파우치들을 박스에 담고 라벨링.

### 비정상 신호
- 외포장 미완료 시 일별실적에 연하늘색 표시
- 박스 무게 불일치 → 내용물 확인

## 7. 전체 시점에서 보는 시그널

### "수율이 낮다"의 진짜 의미

| 어디서 떨어졌나? | 진짜 원인 |
|---|---|
| **전처리에서 비가식부 多** | **원물 품질 불량** (가장 가능성 높음) |
| **자숙에서 손실 多** | 해동/자숙 시간 또는 온도 이슈 |
| **파쇄에서 손실 多** | 자숙 또는 전처리 단계 누락 |
| **포장에서 불량 多** | 기계 점검, 작업자, 환경 이슈 |

**핵심**: 수율 저하의 90% 이상은 **상위 공정**에서 원인이 시작됨. AI는 항상 **상류 공정부터 역추적**해야 함.

---

## 8. 공급처/로트 추적

원물 입고 시 `barcode.importCode`에 다음 정보 포함:
- 공급처 코드
- 입고 일자
- 부위 코드
- 박스 일련번호

비가식부 다량 발생 시 같은 importCode prefix의 박스들을 추적하면 **공급처/로트별 품질 패턴** 발견 가능.

---

## 핵심 룰 요약

1. 수율 저하 → **상류 공정부터** 원인 추적
2. 전처리 비가식부 多 → **원물 품질 의심 1순위**
3. 자숙 손실 多 → 해동/자숙 시간/온도 점검
4. 부위별 특성 다름 (홍두깨는 원래 비가식부 많음)
5. 같은 패턴이 같은 입고일/공급처에 집중되면 → **공급처 협의 필요**
