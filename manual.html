직원 분류 (role) 2026-05-14

각 직원에 role(역할) 정보가 있습니다:

• production (생산직, 기본값): 작업인원 카운트에 포함

• qc (품질관리): 출퇴근 화면엔 정상 표시되지만 작업인원 카운트에서 제외

• manager (관리직): 동일하게 제외

월간 생산 일보의 "작업인원" 수치는 production만 카운트합니다. 직원 추가 시 기본은 production이며, QC/관리직은 role 변경 필요.

💡 직원 마스터는 모든 디바이스 공유(Firestore). PC에서 추가한 직원이 태블릿/모바일에서도 자동으로 보입니다.

입고관리 (1공장 / 2공장 분리) 2026-05-14 개편

외부 입고 + 공장간 이동 + 월별 이력 조회

1

화면 구조

입고관리 페이지는 2개 서브탭으로 나뉩니다: 🏭 2공장 / 📦 1공장. 각 탭마다 동일한 구성 (재고카드 + 외부 입고폼 + 입고 이력 + 출고 이력 + 공장간 이동폼).

2

재고 카드 보는 법

부위별(설도/우둔/홍두깨)로 카드 표시. 잔여 박스 = 입고 + 들어온 이동 − 나간 이동 − 사용량. 박스당 20kg 고정으로 "약 N kg" 자동 환산. 해동중이면 "해동중 N박스" 표시 + 화살표로 오늘→내일 비교.

3

외부 입고 추가

입고일 / 부위 / 박스 수 / 메모(원산지·로트번호 등) 입력 후 저장.

4

공장간 이동

"1공장으로 이동" / "2공장으로 이동" 폼에서 방향·부위·박스 수·사유 입력. 저장 시 자동으로 양쪽 이력(나간 쪽 출고이력 + 받은 쪽 입고이력)에 동시 등록됩니다.

5

월별 필터 (이력 조회)

이력 카드 위에 월 선택기:

◀  2026년 5월 [이번달]  📅 ▶

• ◀▶: 한 달씩 이동 (이번달, 4월, 3월...)

• 📅: 달력 팝업으로 직접 년/월 선택 — 데이터 누적되면 화살표만 누르기 힘드니까 점프용

• 재고 카드는 항상 최신 — 월 선택과 무관

• 1공장/2공장 탭 각자 독립 — 한 쪽 바꿔도 다른 쪽 영향 없음

AI 분석 + 챗봇 2026-05-13 신설

기간 분석 + 자연어 챗봇 (Gemini Flash)

1

두 가지 탭

AI 분석 페이지에 2개 탭: 📊 기간 분석 (기간을 지정하면 AI가 데이터를 분석하여 종합 리포트 작성) / 💬 챗봇 (자유로운 자연어 질의응답).

2

챗봇 사용법

질문 예시:

• "어제 원육수율이 평소보다 낮은 이유 분석해줘"
• "5월 홍두깨 비가식부율 추이 알려줘"
• "최근 일주일 동안 가장 생산성 좋은 날은?"
• "FC 장조림 3KG 만들 때 보통 며칠 걸려?"

최근 60일 데이터를 자동 fetch해서 답변. 대화 이력은 모든 디바이스에 공유 저장.

3

도메인 지식

AI는 다음 도메인 지식 문서를 자동으로 참조합니다 (GitHub 자동 fetch):

• 공정 원리: 각 공정의 정상 범위, 비정상 신호, 상류 추적 흐름

• 진단 룰북: 증상→원인 패턴 7가지

• 공장 특수 정보: 부위/제품/레토르트 운영 룰

답변이 부정확하면 관리자가 도메인 지식 MD를 보강하여 개선할 수 있습니다.

4

한도 / 비용

일 1500회 호출. 평소 운영하면 무료 범위. 5/13 처음 운영 시 일 5만회 초과로 ₩950 청구된 적 있으나, 50배 성능 최적화 적용 후 정상 운영.
